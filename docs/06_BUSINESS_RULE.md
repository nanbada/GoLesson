# 06. BUSINESS RULE — 업무 규칙

번호는 BR-xxx. 구현·QA 시 이 문서가 판정 기준 (PRD와 충돌 시 PRD 우선).

## 진도 (BR-100)

- **BR-101** 진도는 로그다. `lesson_progress`에 구간(from→to)으로 누적 기록하며 현재값 덮어쓰기 금지.
- **BR-102** 중복·역행 **허용 + 경고**. to ≤ 직전 to이면 "이전 진도(p42)보다 낮거나 같습니다 — 복습이면 그대로 저장하세요" 표시. 차단하지 않는다(복습·재학습은 정상 운영).
- **BR-103** from 기본값 = 해당 student_textbook의 last_position (첫 기록이면 0 또는 직접 입력).
- **BR-104** from/to가 역순이면 저장 시 작은 값→큰 값으로 자동 정렬한다. 예: 42→38 입력은 38→42 로그로 저장한다.
- **BR-105** 교재 완료: total_units가 있고 to ≥ total_units → 완료 확인 다이얼로그 → status=completed, completed_on 기록. 자동 완료 처리 금지(반드시 확인).
- **BR-106** 완료된 교재에는 진도 입력 불가. 이어서 배정된 다음 교재로 입력.
- **BR-107** 진도 단위는 교재당 1개(unit_label). 페이지·단원 혼용 교재는 대표 단위 1개를 정하고 나머지는 memo에 기재.

## 수업 (BR-200)

- **BR-201** 수업 = 학생×과목×1회. 같은 날 같은 과목 2회(보강)도 별개 lesson.
- **BR-202** [수업 시작] 시각과 [완료] 시각은 자동 기록. 수업시간(분) = ended−started, 표시용.
- **BR-203** 시작 없이 기록만 하는 경우(빠른 입력 등) started/ended는 null 허용, status=done.
- **BR-204** 취소된 수업(canceled)은 통계(출석률·완료율)에서 제외.
- **BR-205** 완료된 수업의 수정은 허용하되 audits에 전후 기록(REQ-1003).

## 과제 (BR-300)

- **BR-301** kind 기본값 `in_class`(학원 내 과제시간). `take_home`은 명시 선택.
- **BR-302** in_class 과제는 당일 수업 화면에서 체크(checked_lesson_id = 당일 lesson).
- **BR-303** take_home 과제는 다음 수업 시작 화면에 자동 표시되고 그 수업에서 체크.
- **BR-304** 미체크(assigned) 과제는 체크될 때까지 다음 수업마다 계속 표시. 자동 소멸 금지.
- **BR-305** 미완료 과제의 재부여 = 새 homework 행 생성 (기존 행 재사용 금지 — 이력 보존).
- **BR-306** 완료율 = done + 0.5×partial ÷ (assigned 제외 전체 체크된 건). 미체크 건은 분모 제외.

## 리포트 (BR-400)

- **BR-401** 상태 흐름: draft → ready(발송 승인) → sent. 어떤 리포트도 검토 없이 발송 불가.
- **BR-402** 통계(출석·진도·과제)는 코드가 계산한 사실 데이터 — AI가 수치를 생성·수정하지 않는다.
- **BR-403** 출석 표기: 기간 내 예정 수업 수(schedule_slots 발생 횟수) 대비 실시 수업 수(lessons done) + 등원 기록(attendance IN). 비율이 아닌 "n회 중 m회" 표기.
- **BR-404** 과목별 의견: 기간 내 comments를 과목별로 수집 → AI 다듬기 → 초안. 코멘트가 없는 과목은 "의견 없음"이 아니라 해당 섹션 생략.
- **BR-405** sent 후 body 불변. 내용 정정은 새 리포트 생성 → 재발송.
- **BR-406** 같은 학생·기간 draft 중복 생성 금지(기존 draft 갱신).

## 발송 (BR-500)

- **BR-501** 발송 대상 = is_primary=true & notify_enabled=true 학부모 1명. 없으면 발송 등록 자체 불가(422).
- **BR-502** dedupe_key = `report:{id}:v{n}`, unique. 같은 버전 중복 등록·중복 발송 불가.
- **BR-503** 재발송은 강사의 명시적 액션으로만(버전 증가). 시스템 자동 재발송 금지 — GoAlimi의 timeout 재시도 금지 원칙과 동일 사상(이중발송 방지).
- **BR-504** Bridge 발송 시간창 09~21시. 그 외 시간의 pending은 다음 시간창까지 대기(실패 아님). GoAlimi의 6~22시 규칙이 최종 방어선.
- **BR-505** 발송 실패는 사유와 함께 강사에게 보이고, 처리는 강사 판단(재전송/포기).
- **BR-506** 실패 상태는 **outbox가 소유**한다. reports.status는 draft/ready/sent 3개뿐이며 발송 실패해도 ready로 남는다(재발송 대상). 화면의 실패 배지·이력은 outbox 기준(dedupe_key 버전 포함).
- *설계 결정* — ready 상태 리포트의 본문은 발송(sent) 전까지 클라이언트가 수정할 수 있고, draft→ready 승인도 클라이언트 전환이다. 이 검토는 동일 강사의 자기 검토이므로(1인 승인 워크플로), 서버 측 승인 강제·ready 본문 잠금은 넣지 않는다(과설계). sent 이후 불변은 DB 트리거가 강제한다.

## 수강료 (BR-600)

- **BR-601** 결제 1건 = 결제일+수단+메모, 금액은 항목(과목·금액) 리스트. 과목 항목은 '영어'·'수학' 외 자유 텍스트 허용(교재비 등).
- **BR-602** 환불 = 음수 금액 항목. 집계는 단순 합산(음수 자동 반영).
- **BR-603** 수정·삭제는 가능하되 audits 기록 필수. 미납·잔액 개념 없음(Non-goal).

## 학생·마스터 데이터 (BR-700)

- **BR-701** 학생·학부모의 생성·수정·비활성화는 GoAlimi에서만. GoLesson은 읽기 전용 사본.
- **BR-702** GoAlimi에서 비활성화된 학생: 목록 제외, 기록은 전부 보존·조회 가능. 스케줄은 자동 비표시(active 조인).
- **BR-703** 동기화 충돌 시 GoAlimi 값이 항상 이긴다(단방향 upsert).
- **BR-704** GoLesson 쪽 학습 데이터(교재 배정·스케줄)는 GoAlimi로 역동기화하지 않는다.

## 계정 (BR-800)

- **BR-801** 계정은 관리자가 Supabase에서 사전 등록(5명 미만). 자가 가입 없음.
- **BR-802** 파일럿에서는 owner/teacher 권한 차이 없음(표시만). 모든 강사가 모든 학생 데이터 접근 가능.
- **BR-803** 동일 계정 다기기 동시 세션 허용 (Supabase Auth 기본 동작 유지 — 단일 세션 강제 설정 금지). 같은 학생 기록을 두 기기가 동시에 수정하는 경우는 후저장 우선(last-write-wins)으로 수용, 별도 잠금 없음(5명 규모).
