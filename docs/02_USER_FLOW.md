# 02. USER FLOW — 사용자 흐름

화면 상세는 03_UI_SPEC, 규칙은 06_BUSINESS_RULE 참조.

## F1. 일일 수업 (가장 빈번 — 하루 수십 회)

```
앱 열기 (PWA)
  → [오늘] 화면: 오늘 요일 스케줄 시간순 표시
  → 학생 카드 탭
  → 수업 화면:
       상단: 직전 수업 요약 (진도 p38→42 · 과제 결과 · 코멘트)
       미체크 take_home 과제 있으면 → 완료/부분/미완료 체크 (+코멘트)
  → [수업 시작] (시작시각 자동)
  → 수업 진행 (앱 조작 없음)
  → 진도 입력: from 자동(직전 to) → to만 입력  ※역행/동일 시 경고, 저장은 가능
  → 과제 입력: 설명 + kind(학원내/집) — 기본값 학원내
  → (학원 내 과제시간 진행) → 과제 체크 + 코멘트
  → [완료] (종료시각·수업시간 자동)
  → [오늘]로 복귀, 해당 카드 '완료' 표시
```

예외: 보강 수업 → [오늘] 하단 [+ 수업 추가] → 학생·과목 선택 → 동일 흐름.
예외: 결석 → 카드의 [취소] → 사유 선택(결석/휴원/기타) → canceled 기록.

## F2. 텍스트 일괄 입력 (수기 메모 이관용)

```
[빠른 입력] 화면
  → 텍스트 여러 줄 붙여넣기 (또는 직접 타이핑)
     예) "민수 영어 브릭스 38-42 숙제 43~45 독해 좋아짐
          서연 수학 쎈 3단원 완료 계산 실수 잦음"
  → [파싱]
  → 줄별 결과 카드: 학생|과목|교재|진도|과제|코멘트  (방식 배지: regex/AI)
     매칭 실패 필드는 빨간 표시 + 드롭다운 수동 지정
  → 카드 수정 (필요시)
  → [일괄 저장] → 각 줄이 lesson(당일)+진도+과제+코멘트로 저장
```

규칙: 미리보기 없이 저장 금지(REQ-504). 당일 이미 수업 기록이 있으면 해당 lesson에 병합.

## F3. 학부모 리포트 (2주/월 1회)

```
[리포트] 화면
  → 기간 선택 (최근 2주 / 이번 달 / 커스텀)
  → 학생 다중 선택 (기본: 활성 전체)
  → [생성] → 학생별 draft 생성
       통계(코드): 출석 n/m · 과목별 진도 시작→끝 · 과제 완료율
       의견(AI): 기간 내 코멘트 → 과목별 학부모용 1문단 초안
  → 학생별 검토: 본문 직접 수정 가능
  → [발송 승인] (ready) → [발송] → outbox 등록
  → 발송 현황: pending → sent(시각) / failed(사유 + [재전송])
```

실제 카톡 도달: 학원 PC Bridge가 09~21시에 폴링·발송 (F5).

## F4. 수강료 기록

```
[수강료] 화면
  → [+ 결제 입력]: 학생 → 날짜(기본 오늘) → 항목 추가(과목·금액) 반복
     → 수단(카드/현금/계좌이체/기타) → 메모 → [저장]
  → 월별 탭: 월 선택 → 총액·수단별·과목별 합계 + 건별 목록
  → 건 탭 → 수정/삭제 (audits 기록) · 환불은 음수 금액 항목
```

## F5. 발송 파이프라인 (시스템 흐름, 강사 비노출)

```
[발송] 클릭 → outbox(pending, dedupe_key)
  → Bridge(학원 PC, 60초 폴링, 09~21시): pending 인출 → processing
  → GoAlimi POST /api/notify/custom (localhost)
  → GoAlimi 직렬 큐 → 카카오톡 PC 자동화 → 학부모 카톡
  → Bridge가 GoAlimi 상태 확인 → outbox sent/failed 반영 → 화면 표시
실패 시: failed + 사유 → 강사가 [재전송] → 새 pending
```

```

```
