# 03. UI SPEC — 화면 명세

## 0. 공통 원칙

| 원칙 | 규칙 |
|---|---|
| One Thumb | 폰 세로 한 손 조작. 주 액션 버튼은 화면 하단 고정 |
| 한 화면 한 작업 | 화면당 주 액션 1개. 탭 수 최소화 |
| 버튼 | 최소 44×44px, 주 액션은 전폭(full-width) |
| 반응형 | 320px~ 세로 우선. 패드/PC는 같은 레이아웃의 max-width 640px 중앙 정렬 + 목록형 화면만 2컬럼 허용 |
| 입력 보존 | 모든 폼은 draft 자동 보존(localStorage) — 이탈·오프라인 대비 |
| 숫자 입력 | 진도·금액은 `inputmode="numeric"` 숫자 키패드 |
| 상태 색 | 대기 회색 · 진행중 파랑 · 완료 초록 · 실패/경고 빨강 |
| 내비게이션 | 하단 탭바 4개: 오늘 · 학생 · 빠른입력 · 더보기(리포트·수강료·교재·발송현황·GoAlimi 바로가기·설정). 2주/월 단위 리포트는 일일 핵심 동선에서 제외 |

## 1. 로그인

이메일+비밀번호 (Supabase Auth). 세션 자동 갱신으로 기기마다 최초 1회 로그인이 기본이며 브라우저 비밀번호 저장을 허용한다. 가입 링크 없음(계정은 관리자 사전 등록, 5명 미만). 동일 계정으로 PC·패드·폰 동시 접속 가능(REQ-909). 전화번호 로그인은 전체 번호의 확인·운영이 필요하고 SMS 복구 비용이 생기므로 MVP에서 사용하지 않는다.

## 2. 오늘 (홈, 기본 진입)

```
┌──────────────────────────┐
│ 7월 3일 (금)   [+ 수업 추가] │
│ ⚠ 동기화 26시간 전 (조건부)   │
├──────────────────────────┤
│ 15:00 · 5명 · 코칭 2/5      │
│ 40분 수업 + 40분 과제·보강   │
│ 김민수 영어 · 오늘 Bricks 42→46 │
│ 과제 완료             [코칭 완료] │
│ 이서연 수학 · 직전 쎈 120까지   │
│ 미체크 1건              [코칭 전] │
└──────────────────────────┘
```

- 데이터: schedule_slots(오늘 요일) + 당일 lessons 병합. 같은 시작 시각을 한 운영 블록으로 파생 그룹화하고 시간순 표시. 학년·과목·레벨은 그룹 키가 아니다.
- 블록 요소: 시각·학생 수·코칭 완료/전체·`duration_min`분 수업 + 40분 과제·보강 안내.
- 학생 행: 이름·과목·코칭 전/중/완료·시작 전에는 직전 진도/미체크 과제, 기록 후에는 오늘 진도/오늘 과제 결과.
- 카드 탭 → 수업 화면. 완료 카드 탭 → 수업 화면(수정 모드).
- 빈 상태: "오늘 수업이 없습니다" + [+ 수업 추가].

## 3. 수업 화면 (핵심 — 30초 목표)

```
┌──────────────────────────┐
│ 김민수 · 영어 · 15:00       │
│ 지난 수업(6/30): p38→42    │
│  과제 완료 · "독해 좋아짐"    │
├──────────────────────────┤
│ ▸ 미체크 과제 (있을 때만)     │
│  단어 Day3  [완료|부분|미완료] │
│  코멘트 ____________       │
├──────────────────────────┤
│ 진도  Bricks Reading (페이지)│
│  42 → [   ]  ※숫자만 입력   │
│  (⚠ 이전 진도 이하 시 경고문)  │
│  [+ 다른 교재]             │
├──────────────────────────┤
│ 과제  [+ 추가]             │
│  ____________ [학원내|집]   │
│  결과 [완료|부분|미완료] 코멘트 │
├──────────────────────────┤
│ 코멘트(선택) ____________    │
├──────────────────────────┤
│ [코칭 기록 완료] (하단 고정)   │
└──────────────────────────┘
```

- 진입 시: 미시작이면 [1:1 코칭 시작] 버튼이 최상단, 탭 시 학생별 started_at 기록 후 폼 노출. lesson 상태는 오늘 블록의 코칭 전/중/완료 현황으로 사용한다.
- 진도 from은 자동(해당 교재 last_position), to만 입력. 단위 라벨은 교재의 unit_label.
- 과제 kind 기본값 `학원내`. 학원내 과제는 같은 화면에서 즉시 결과 체크 가능.
- [코칭 기록 완료] → ended_at 기록 → 오늘 화면 복귀 후 블록 완료 수와 오늘 진도·과제 요약 갱신. 이 시각은 80분 블록 전체가 아니라 학생별 코칭 기록 동작의 시작·종료다.

## 4. 학생 목록 / 상세

- 목록: 검색(이름) + 활성 필터. 행: 이름(동명이인 시 학년 병기)·과목 배지.
- 상세 탭 4개:
  - **요약**: 과목·스케줄·배정 교재(현재 위치/총량, 진행 바)·미체크 과제·최근 코멘트
  - **진도**: 교재별 히스토리(날짜 from→to) + 최근 5회 추이 미니차트 · [교재 배정] [완료 처리→다음 교재]
  - **과제**: 기간별 목록 + 완료율
  - **결제**: 이력 목록 + [+ 결제 입력]
- 학생 정보 수정 버튼 없음 — "학생 정보는 GoAlimi에서 관리합니다" 안내(REQ-103).
- 스케줄 편집: 요약 탭에서 과목별 요일·시각 편집 (REQ-201).

## 5. 빠른 입력 (파싱 도구)

```
┌──────────────────────────┐
│ 텍스트 붙여넣기              │
│ ┌──────────────────────┐ │
│ │민수 영어 브릭스 38-42 ...│ │
│ └──────────────────────┘ │
│ [파싱하기]                 │
├── 결과 카드 (줄별) ─────────┤
│ ✓ 김민수|영어|Bricks|38→42 │
│   과제: 43~45 · "독해 좋아짐"│
│   (regex)          [수정]  │
│ ✗ 학생 인식 실패: "슬기"      │
│   [학생 선택▾]  (AI)       │
├──────────────────────────┤
│ [3건 일괄 저장]             │
└──────────────────────────┘
```

- 오류 카드가 하나라도 미해결이면 해당 카드만 제외하고 저장할지 확인.
- 저장 성공 후 원문·결과를 parse_logs에 confirmed로 기록.

## 6. 리포트

- 목록: 기간·학생·상태(draft/ready/sent) 필터. 발송 실패 배지는 outbox 상태에서 조인해 표시(BR-506 — reports에 failed 상태 없음).
- 생성: 기간 프리셋(최근 2주/이번 달/커스텀) + 학생 다중 선택 → [생성].
- 상세(검토):
  - 상단: 통계 카드(출석 n/m · 과목별 진도 · 과제 완료율) — 수정 불가(사실 데이터)
  - 본문: 전체 텍스트 편집 가능 (템플릿+AI 초안이 미리 채워짐), 글자 수 카운터(600~900자 가이드)
  - 버튼: draft → [발송 승인] → ready → [카카오톡 발송] / sent → [재발송]
  - sent 이후 본문 읽기 전용(REQ-1004).
- 수신자 표시: "발송 대상: 김민수 모(카톡명 ○○○)". 대표 학부모 없으면 발송 버튼 비활성+사유.

## 7. 발송 현황 (더보기)

outbox 목록: 시각·학생·유형·상태·오류사유·[재전송]. 상단 요약(대기 n·성공 n·실패 n).
Bridge 최근 폴링 시각(app_settings.bridge_last_poll_at) 표시 — 30분 초과 시 "학원 PC 연결 확인 필요" 경고.

## 8. 수강료 (더보기)

- 입력 폼: 학생 선택 → 날짜 → 항목 리스트(과목 선택+금액) [+ 항목] → 수단 세그먼트 → 메모 → [저장].
- 월별 뷰: 월 네비게이션 ← 7월 → · 합계 카드(총액/카드/현금/이체) · 건별 리스트(탭→수정).

## 9. 교재 관리 (더보기)

목록(과목 필터) → 등록/수정 폼: 과목·제목·출판사·진도 단위 라벨(기본 '페이지')·총량(선택)·별칭(쉼표 구분).
삭제 버튼 없음 — 비활성화만(REQ-1002).

## 10. GoAlimi 바로가기 (더보기, REQ-910)

더보기 목록의 "GoAlimi 출결 관리" 항목 → `app_settings.goalimi_admin_url`을 새 탭(`target="_blank" rel="noopener"`)으로 오픈.
항목 부제: "학원 네트워크에서만 열립니다". iframe 임베드 금지(mixed content — 11_STUDY §8), 도달 여부 자동 감지 시도 금지(동일 사유).

## 11. 설정 (더보기)

내 계정(이름·로그아웃) · 강사 목록(역할 표시) · 학원명·리포트 문구 편집(app_settings: 인사말/맺음말) · GoAlimi 주소(goalimi_admin_url) · 동기화 상태(최근 sync 시각).
